# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.x   | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in CreativeForge AI Studio, please report it responsibly:

1. **Do NOT** open a public GitHub issue for security vulnerabilities.
2. Email the maintainer directly at **22103330@iubat.edu** with:
   - A description of the vulnerability
   - Steps to reproduce
   - Potential impact assessment
   - Suggested fix (if any)

3. You will receive an acknowledgment within 48 hours.
4. The issue will be investigated and addressed as quickly as possible.
5. You will be credited for the discovery (unless you prefer to remain anonymous).

## Security Best Practices

When deploying CreativeForge AI Studio:

- Never commit `.env` files or secrets to the repository
- Use strong, unique API keys for all AI model providers
- Enable HTTPS in production
- Keep all dependencies updated
- Use the principle of least privilege for database access
- Regularly rotate API keys and credentials
